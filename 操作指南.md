## 环境准备

### 系统要求

- Node.js 18.0 或更高版本
- npm 8.0 或更高版本

### 检查环境

```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version
```

如果未安装 Node.js，请访问 https://nodejs.org 下载安装。

## 安装步骤

### 1. 通过 npm 使用

```bash
npx -y bigopen-llm-search
```

MCP 客户端推荐直接使用 `npx -y bigopen-llm-search`，不需要克隆仓库。

### 2. 本地开发

需要修改源码时再克隆仓库：

```bash
git clone <repository-url>
cd BigOpenLLMSearch
npm install
npm run build
```

构建成功后，会在 `dist` 目录生成 JavaScript 文件。

## 配置搜索引擎

### 支持的搜索引擎

| 搜索引擎 | 是否需要 API Key | 获取地址 |
|----------|------------------|----------|
| SearXNG | ✅ 需要自建 | 推荐默认搜索引擎 |
| DuckDuckGo | ❌ 不需要 | 未配置 SearXNG 时降级使用 |
| Bing | ✅ 需要 | https://www.microsoft.com/en-us/bing/apis/bing-web-search-api |
| Google | ✅ 需要 | https://developers.google.com/custom-search/v1/overview |
| Brave | ✅ 需要 | https://brave.com/search/api/ |
| SerpApi | ✅ 需要 | https://serpapi.com/ |

### 配置方法

#### 方法一：使用 .env 文件（推荐）

```bash
# 复制配置模板
cp .env.example .env

# 编辑配置文件
nano .env
```

在 `.env` 文件中填入你的搜索引擎配置。推荐使用 SearXNG，可以连接现有实例，也可以让 MCP 自动拉起本地 Docker 容器：

```env
# 方案一：已有 SearXNG 实例；配置后默认使用 SearXNG
SEARXNG_URL=http://127.0.0.1:8888

# 方案二：没有现成实例时，让 MCP 在首次搜索前自动拉起本地 Docker 容器
SEARXNG_AUTO_START=true
SEARXNG_DOCKER_PORT=8888

# 可选：显式指定默认搜索引擎
DEFAULT_SEARCH_ENGINE=searxng

# Bing 搜索 API
BING_API_KEY=你的Bing密钥

# Google 自定义搜索 API
GOOGLE_API_KEY=你的Google密钥
GOOGLE_SEARCH_ENGINE_ID=你的搜索引擎ID

# Brave 搜索 API
BRAVE_API_KEY=你的Brave密钥

# SerpApi
SERPAPI_API_KEY=你的SerpApi密钥

# 可选：AI 答案生成（OpenAI 兼容接口）
LLM_API_KEY=你的OpenAI兼容接口密钥
LLM_MODEL=你的模型名
LLM_BASE_URL=https://你的OpenAI兼容接口/v1
```

`LLM_API_KEY` 和 `LLM_MODEL` 同时配置后，`include_answer` 会使用 LLM 基于搜索来源生成答案；没有配置时，服务会自动返回抽取式摘要，不影响搜索、提取、爬取和站点地图能力。

`SEARXNG_AUTO_START=true` 需要本机已安装并启动 Docker。服务会在首次需要 SearXNG 搜索前创建或复用名为 `bigopen-llm-search-searxng` 的容器，并在 `~/.bigopen-llm-search/searxng/config/settings.yml` 生成启用 `json` 输出的 SearXNG 配置。已有自定义 `settings.yml` 时不会覆盖，需要自行确认 `search.formats` 包含 `json`。

`topic` 的查询扩展词与重排词在 `config/search-topics.json` 中维护。需要调整新闻、金融等主题策略时，优先改配置文件，不要在代码里针对具体搜索关键词写特例。

#### 方法二：使用环境变量

```bash
export BING_API_KEY=你的Bing密钥
export GOOGLE_API_KEY=你的Google密钥
export GOOGLE_SEARCH_ENGINE_ID=你的搜索引擎ID
export SEARXNG_URL=http://127.0.0.1:8888
export SEARXNG_AUTO_START=true
```

### API Key 获取指南

#### Bing API Key

1. 访问 https://portal.azure.com
2. 创建 "Bing Search v7" 资源
3. 在 "Keys and Endpoint" 中获取 API Key

#### Google Custom Search API

1. 访问 https://console.cloud.google.com
2. 启用 "Custom Search API"
3. 创建 API Key
4. 访问 https://programmablesearchengine.google.com 创建搜索引擎
5. 获取搜索引擎 ID

#### Brave Search API

1. 访问 https://brave.com/search/api/
2. 注册并选择套餐
3. 获取 API Key

## 客户端配置

### Claude Desktop 配置

#### macOS

编辑文件：`~/Library/Application Support/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "bigopen-llm-search": {
      "command": "npx",
      "args": ["-y", "bigopen-llm-search"],
      "env": {
        "SEARXNG_AUTO_START": "true"
      }
    }
  }
}
```

#### Windows

编辑文件：`%APPDATA%\Claude\claude_desktop_config.json`

### Cursor 配置

在项目根目录创建 `.cursor/mcp.json`：

```json
{
  "mcpServers": {
    "web-search": {
      "command": "npx",
      "args": ["-y", "bigopen-llm-search"],
      "env": {
        "SEARXNG_AUTO_START": "true"
      }
    }
  }
}
```

### 重启客户端

配置完成后，重启 Claude Desktop 或 Cursor 使配置生效。

## 工具使用示例

### 示例 1：基础网页搜索

**请求：**
```
请搜索"2024年人工智能最新进展"
```

**LLM 调用：**
```json
{
  "tool": "web_search",
  "arguments": {
    "query": "2024年人工智能最新进展",
    "max_results": 5
  }
}
```

### 示例 2：新闻搜索

**请求：**
```
搜索最近一周关于 ChatGPT 的新闻
```

**LLM 调用：**
```json
{
  "tool": "web_search",
  "arguments": {
    "query": "ChatGPT",
    "topic": "news",
    "time_range": "week",
    "max_results": 10
  }
}
```

### 示例 3：提取网页内容

**请求：**
```
帮我提取这篇文章的内容：https://example.com/article
```

**LLM 调用：**
```json
{
  "tool": "web_extract",
  "arguments": {
    "urls": "https://example.com/article",
    "format": "markdown"
  }
}
```

### 示例 4：爬取网站

**请求：**
```
爬取 https://docs.example.com 网站的所有 API 文档
```

**LLM 调用：**
```json
{
  "tool": "web_crawl",
  "arguments": {
    "url": "https://docs.example.com",
    "instructions": "查找所有 API 文档页面",
    "select_paths": ["/api", "/docs"],
    "max_depth": 3
  }
}
```

### 示例 5：生成网站地图

**请求：**
```
帮我分析 https://example.com 的网站结构
```

**LLM 调用：**
```json
{
  "tool": "web_map",
  "arguments": {
    "url": "https://example.com",
    "max_depth": 3,
    "limit": 100
  }
}
```

### 示例 6：语义搜索

**请求：**
```
用语义搜索找关于量子计算的最新突破
```

**LLM 调用：**
```json
{
  "tool": "web_search",
  "arguments": {
    "query": "量子计算最新突破",
    "semantic": true,
    "max_results": 10
  }
}
```

## 常见问题

### Q1: 启动时报错 "Cannot find module"

**解决方案：**
```bash
# 重新安装依赖
rm -rf node_modules
npm install

# 重新构建
npm run build
```

### Q2: 搜索没有结果

**可能原因：**
1. 未配置任何搜索引擎 API Key
2. API Key 无效或已过期
3. 网络连接问题

**解决方案：**
1. 检查 `.env` 文件配置
2. 验证 API Key 是否有效
3. 检查网络连接

### Q3: Claude Desktop 无法连接 MCP 服务器

**解决方案：**
1. 检查配置文件路径是否正确
2. 确保 `dist/index.js` 文件存在
3. 重启 Claude Desktop
4. 检查 Node.js 路径：`which node`

### Q4: 搜索速度慢

**解决方案：**
1. 使用 `fast` 或 `ultra-fast` 搜索深度
2. 减少 `max_results` 数量
3. 配置更快的搜索引擎（如 Bing）

### Q5: 如何指定搜索引擎？

`web_search` 默认优先使用 `DEFAULT_SEARCH_ENGINE`。未显式配置时，如果存在 `SEARXNG_URL` 或 `SEARXNG_AUTO_START=true`，默认使用 SearXNG；否则降级为 DuckDuckGo。你可以在请求中指定单个搜索引擎：

```json
{
  "tool": "web_search",
  "arguments": {
    "query": "你的查询",
    "engine": "bing"
  }
}
```

### Q6: 如何更新到最新版本？

```bash
npm update -g bigopen-llm-search
```

如果 MCP 客户端使用 `npx -y bigopen-llm-search`，重启客户端后会按 npm 缓存策略拉取可用版本；需要强制刷新时可以清理 npm 缓存或指定版本号。
